# スプレッドシート・ワークショップ・ガイド (Live Session Guide)

本ドキュメントは、顧客と画面共有をしながら `gcp_foundations.xlsx` (SSOT) を完成させるためのファシリテーション・ガイドです。

単に Excel を埋めるだけでなく、顧客のビジネス要件を GCP の構造にどう落とし込むかの意思決定をサポートします。

______________________________________________________________________

## 1. セッションの進め方 (Agenda)

以下の順番で進めることを推奨します。

1. **階層構造の設計 (`resources` シート: フォルダ)**
1. **ネットワーク設計 (`shared_vpc_subnets` シート)**
1. **セキュリティ統制の設計 (`vpc_sc_perimeters`, `org_policies` シート)**
1. **プロジェクトの定義 (`resources` シート: プロジェクト)**

______________________________________________________________________

## 2. 各シートの決定ポイントと問いかけ

### 2.1 フォルダ構造の設計

**問いかけ**: 「開発・本番で環境を分けますか？」「部門ごとに管理権限を分けますか？」

- **推奨事項**: 最初はシンプルに `shared`, `production`, `development` の 3 階層から始めることを勧めます。
- **Excel への入力**: `folder` 行を追加し、親子関係を定義します。

### 2.2 ネットワークの設計

**問いかけ**: 「ネットワーク管理を中央に集約しますか？それとも各プロジェクトで自由に作らせますか？」

- **意思決定のヒント**: Shared VPC を使う（中央管理）ことで、IP アドレスの枯渇防止とセキュリティ統制が容易になります。
- **Excel への入力**: `shared_vpc_subnets` シートに、各環境（prod/dev）で必要なサブネットを定義します。

### 2.3 組織ポリシー (ガードレール) の設計

**問いかけ**: 「最初から厳格な制限をかけますか？それとも既存資産の移行が終わってから締めますか？」

- **重要**: `common.tfvars` の `enable_org_policies` スイッチについて説明し、最初は `FALSE` で進める選択肢を提示してください。
- **主なポリシー例**:
  - `compute.disableExternalIPProxy`: 外部 IP の付与を禁止（セキュリティ向上）。
  - `gcp.resourceLocations`: リソース作成を日本（asia-northeast1）のみに制限（ガバナンス）。

### 2.4 プロジェクトの定義

**問いかけ**: 「どのプロジェクトがどのネットワーク（サブネット）を使い、どのセキュリティ境界に属すべきですか？」

- **Excel への入力**: `resources` シートに `project` 行を追加し、前のステップで決めたサブネット名や境界名を紐付けます。

______________________________________________________________________

## 3. ライブ入力中の注意点

1. **一意な名前の確保**: `subnet_name` や `perimeter_name` は、リポジトリ全体で重複しないようにしてください。
1. **型（True/False）**: Excel の論理値（TRUE/FALSE）として入力されているか確認してください。
1. **命名規則の確認**: プロジェクト名は `prd-app-01` のように、環境と用途がわかる形式を推奨してください。

______________________________________________________________________

## 4. セッションの締めくくり

1. **保存と実行**: Excel を保存し、その場で `make generate` を実行します。
1. **差分の確認**: 実行後、`terraform/4_projects/` などに新しいディレクトリができていることを顧客に見せ、「これがあなたの定義した設計図から自動生成されたコードです」と説明します。
1. **デプロイへの合意**: 構成に間違いがないことを確認し、デプロイ（`make deploy`）のステップへ進みます。
