# 顧客への引き渡し（Handover）手順

このドキュメントは、構築したGCP基盤およびIaCリポジトリを最終的に顧客（本来のインフラ管理者）へ引き渡す際の手順を説明します。

## 1. Git履歴のクリーンアップ

構築中の試行錯誤の履歴や、誤ってコミットされた可能性のある一時的な機密情報を完全に消去するため、Gitの履歴をリセットして真っ新な状態で顧客に渡します。

以下の自動化スクリプトを実行してください。

```bash
bash terraform/scripts/handover.sh
```

このスクリプトは `.git` フォルダを削除し、現在の最新状態で `Initial commit` を作成し直します。その後、作成されたリポジトリを顧客指定のGitホスティングサービス（GitHub, GitLabなど）へPushしてください。

## 2. GCP権限の移譲 (IAM)

Terraformの実行基盤となる `0_bootstrap` で作成されたリソースの権限を顧客の管理者に移譲し、構築者（ベンダーや初期セットアップ者）の権限を剥奪します。

### ステップ 1: 顧客管理者への権限付与

GCPコンソールを使用して、顧客の管理者ユーザー（またはGoogleグループ）に対し、以下のロールを付与します。

- 組織に対する **組織管理者** (`roles/resourcemanager.organizationAdmin`)
- `*-tfstate` という名前の管理プロジェクトに対する **オーナー** (`roles/owner`)

### ステップ 2: 構築者の権限剥奪

移譲が完了し、顧客自身でGCPコンソールにアクセスできること、および `deploy_all.sh` を実行できることを確認したら、構築者自身のアカウントのIAMバインディング（組織レベル、プロジェクトレベル）を削除して業務を完了します。
