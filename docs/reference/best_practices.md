# ベストプラクティス集 (Best Practices)

本リポジトリでの運用・開発において、推奨されるプラクティスと IAM 権限管理の方針をまとめます。

______________________________________________________________________

## 1. インフラ管理の原則

### 1.1. Terraform 優先 (IaC First)

リソースの作成・変更は、原則として Terraform で行います。

- **推奨順位**: Terraform > gcloud コマンド > Google Cloud コンソール
- **理由**: コンソールでの手作業は再現性がなく、構成の「漂流 (Drift)」を招きます。検証目的でコンソールを使用した場合も、最終的には必ずコードに反映してください。

### 1.2. 信頼できる唯一の情報源 (SSOT)

構成情報は `gcp_foundations.xlsx` で管理します。

- ディレクトリを手動で作成したり、`tfvars` を手書きするのではなく、`make generate` コマンドを使用してコードを生成してください。

## 2. IAM ・権限管理の方針

### 2.1. 最小権限の原則 (Least Privilege)

ユーザーやサービスアカウントには、業務遂行に必要な最小限の権限のみを付与します。

### 2.2. グループベースの管理

個人のメールアドレスに直接権限を付与せず、役割ごとに定義された Google グループ（例：`gcp-security-admins@...`）を介して権限を管理します。

### 2.3. サービスアカウントの借用 (Impersonation)

機密性の高い JSON キーファイルを発行・保持せず、`gcloud` や Terraform からサービスアカウントを「借用」して実行します。これにより、鍵の漏洩リスクを排除し、監査ログに実行者の情報を記録します。

## 3. 管理用 Google グループの定義

| グループ名 | 役割・目的 |
| :--- | :--- |
| **gcp-organization-admins** | 組織管理者。組織に属するすべてのリソースを管理する権限を持ちます。 |
| **gcp-billing-admins** | 請求管理者。請求先アカウントの設定とその使用状況の監視を担当します。 |
| **gcp-vpc-network-admins** | ネットワーク管理者。VPC ネットワーク、サブネット、ファイアウォール ルールの作成を担当します。 |
| **gcp-hybrid-connectivity-admins** | ハイブリッド接続管理者。Cloud VPN やクラウド ルーターなどのネットワーク デバイスの作成を担当します。 |
| **gcp-logging-monitoring-admins** | ロギング / モニタリング管理者。Cloud Logging および Cloud Monitoring のすべての機能にアクセスできます。 |
| **gcp-logging-monitoring-viewers** | ロギング / モニタリング閲覧者。ログおよびモニタリング データの読み取り専用アクセス権を持ちます。 |
| **gcp-security-admins** | セキュリティ管理者。アクセス管理や組織の制約ポリシーなどの組織全体のセキュリティ ポリシーを管理します。 |
| **gcp-developers** | 開発者。アプリケーションの設計、コーディング、テストを担当します。組織全体の閲覧権限を持ちます。 |
| **gcp-devops** | DevOps 従事者。CI/CD パイプライン、システム プロビジョニングをサポートする仕組みを管理します。 |

______________________________________________________________________

## 4. 運用上のヒント

- **デプロイ前の検証**: `terraform plan` および `make opa` を実行し、意図しない変更やポリシー違反がないか必ず確認してください。
- **モジュールのバージョン固定**: 外部モジュールを使用する場合は、コミットハッシュやタグを使用してバージョンを固定し、予期せぬ破壊的変更を防いでください。
