# セキュリティポリシー (Security Policy)

本リポジトリ（GCP Foundations）は、安全なGCP環境を構築・管理するための基盤となるIaCコードを提供しています。インフラストラクチャにおけるセキュリティは最優先事項であり、発見された脆弱性やセキュリティリスクに対する責任ある開示と迅速な対応を求めています。

## サポートされるバージョン
現在、セキュリティ更新の対象となるバージョンは `main` ブランチの最新状態のみです。

## 脆弱性の報告 (Reporting a Vulnerability)

### 連絡先
**重大な脆弱性（APIキーの流出、不正アクセスの可能性、深刻な設計上の欠陥など）を発見した場合、公開のIssueやPull Requestを作成することは絶対に避けてください。**

代わりに、リポジトリの管理者に対してGitHubの「Security Advisories」機能を利用してプライベートに報告を行うか、記載された管理者の連絡先まで直接メールでご連絡ください。

### 報告に含めるべき内容
脆弱性を迅速に調査・修正するため、以下の情報を含めてください。
- 問題の概要と潜在的な影響
- 再現手順（可能な限り詳細に）
- 対象となるTerraformモジュールやスクリプトのパス
- 問題を修正するための提案や緩和策（もしあれば）

### 対応の流れ
報告を受け取った後、以下のステップで対応を行います。
1. **受領の確認:** 報告を受領した旨を速やかに（通常48時間以内に）返信します。
2. **調査と評価:** 問題の深刻度と影響範囲を評価します。
3. **修正と検証:** 問題を解決するためのパッチを作成し、テスト環境で検証を行います。
4. **公開と開示:** 修正がメインブランチにマージされた後、必要に応じてセキュリティアドバイザリを公開し、利用者にアップデートを促します。

## シークレット管理の原則
本リポジトリへの貢献において、以下の点を厳守してください。
- サービスアカウントのJSONキー、APIキー、パスワードなどの認証情報（シークレット）をコード内に直接ハードコードしたり、コミットしたりすることは**絶対に禁止**されています。
- `domain.env` や `*.tfvars` など、特定の環境設定や認証情報を含む可能性のあるファイルは `.gitignore` で管理され、コミットされないようになっています。この構成を変更しないでください。
- 万が一、シークレットをコミットしてしまった場合は、直ちにローカルリポジトリで該当コミットを打ち消し、すでにPushしてしまった場合は該当シークレットの速やかな無効化（ローテーション）を行ってください。
